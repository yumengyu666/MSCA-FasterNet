## 步骤三：改进模块设计（第2-3周）

### 3.1 改进点1：多尺度通道注意力模块（MSCA）

**设计动机**：病虫害场景下病斑大小差异极大（小到3-5像素，大到覆盖半叶），FasterNet原版的3×3 PConv对小病斑感受野不够，对大病斑覆盖不足。

**模块结构**：

```
输入特征 X (C, H, W)
    │
    ├── 分支1（通道注意力）:
    │   AdaptiveAvgPool2d(1) → 1×1 Conv(C→C/r) → ReLU → 1×1 Conv(C/r→C) → Sigmoid
    │   输出: channel_weights (C, 1, 1)
    │
    ├── 分支2（中等尺度空间特征）:
    │   3×3 DepthwiseConv(C, groups=C) → BN → GELU
    │   输出: F_3 (C, H, W)
    │
    ├── 分支3（大尺度空间特征）:
    │   5×5 DepthwiseConv(C, groups=C) → BN → GELU
    │   输出: F_5 (C, H, W)
    │
    ├── 融合: F_fused = F_3 + F_5 （逐元素相加）
    │
    └── 校准: Output = F_fused ⊗ channel_weights （广播相乘）
```

**关键设计参数**：

| 参数 | 值 | 选择理由 |
|------|-----|---------|
| reduction | 16 | SE的标准压缩比，平衡表达力和参数量 |
| DWConv核大小 | 3×3 + 5×5 | 覆盖中等和大尺度；不用1×1因为PConv已有 |
| 激活函数 | GELU | 与FasterNet原版保持一致 |
| 归一化 | BN | 与FasterNet原版保持一致 |
| 融合方式 | 逐元素相加 | 最简单，参数零增加；不拼接避免通道翻倍 |

**插入位置**：仅插入Stage3（dim=160）最后2个Block的PConv之后

**为什么只插Stage3**：
- Stage3有8个Block，特征最丰富，改进空间最大
- Stage1-2特征太浅，改进意义不大
- Stage4只有2个Block且空间分辨率低（7×7），插入收益小
- 只插2个Block控制参数增加量（每个MSCA仅~9K参数）

**参数增加量**：2个MSCA共 ~18K 参数（占原模型3.9M的0.46%）

### 3.2 改进点2：跨层特征融合策略

**设计动机**：原版FasterNet只用Stage4输出做分类。但病虫害识别既需要浅层的纹理细节（病斑颜色、形状），也需要深层的语义信息（病变类型），单一层不够。

**融合结构**：

```
Stage2输出 (80, 28, 28)  → 1×1 Conv(80→160) → AvgPool 2× → (160, 14, 14) ─┐
Stage3输出 (160, 14, 14) → 直接使用                                        → (160, 14, 14) ─┤→ Concat → (480, 14, 14)
Stage4输出 (320, 7, 7)   → 1×1 Conv(320→160) → Upsample 2× → (160, 14, 14) ─┘
                                                                              │
                                                                    1×1 Conv(480→160) + BN + GELU
                                                                              │
                                                                         MSCA(160)
                                                                              │
                                                                    GAP → FC(160→num_classes)
```

**关键设计参数**：

| 参数 | 值 | 选择理由 |
|------|-----|---------|
| 对齐维度 | 160 | 以Stage3通道数为基准，避免信息损失 |
| 对齐空间 | 14×14 | 以Stage3空间尺寸为基准，7×7太小丢失细节 |
| S2下采样方式 | AvgPool 2× | 简单无参数，保留更多信息 |
| S4上采样方式 | Bilinear 2× | 简单无参数，避免转置卷积的棋盘效应 |
| 融合后压缩 | 1×1 Conv 480→160 | 降维+跨通道交互 |
| 融合后校准 | MSCA(160) | 让融合特征重新校准通道权重 |

**为什么选14×14而不是7×7**：
- 7×7太小，只有49个空间位置，病斑细节容易丢失
- 14×14有196个空间位置，保留更多病斑形状信息
- 28×28太大，计算量翻4倍但信息增益有限

**参数增加量**：融合模块共 ~150K 参数（含1×1对齐+压缩+MSCA）

### 3.3 完整模型架构总览

```
输入图像 (3, 224, 224)
    │
    ▼
Embedding: 4×4 Conv, stride=4 → (40, 56, 56)
    │
Stage1: 1× FasterNet Block → (40, 56, 56)          ← 不改动
    │
Merging: 2×2 Conv, stride=2 → (80, 28, 28)
    │
Stage2: 2× FasterNet Block → (80, 28, 28)          ← 保存输出用于融合
    │
Merging: 2×2 Conv, stride=2 → (160, 14, 14)
    │
Stage3: 8× FasterNet Block → (160, 14, 14)         ← 最后2个Block加MSCA；保存输出用于融合
    │
Merging: 2×2 Conv, stride=2 → (320, 7, 7)
    │
Stage4: 2× FasterNet Block → (320, 7, 7)           ← 保存输出用于融合
    │
    ├──→ 跨层融合: Stage2+Stage3+Stage4 → (160, 14, 14)
    │
    ├──→ GAP → (160, 1, 1) → Flatten → (160)
    │
    └──→ FC(160→102) → 输出
```

**总参数量**：~4.07M（原版3.9M + MSCA 18K + 融合150K）
**总FLOPs**：~0.40G（原版0.34G + 改进增加部分）

| 总样本数 | ~75,000 | ~54,000 |
| 划分方式 | 官方train/val/test | 8:1:1随机划分 |
| 输入尺寸 | 224×224 | 224×224 |
| 采样策略 | WeightedRandomSampler | 均匀采样（数据较均衡） |
| 任务类型 | 细粒度分类 | 粗粒度分类 |

### 4.2 训练超参数

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 优化器 | AdamW | 标准选择 |
| 初始学习率 | 4×10⁻⁴ | 比ImageNet的4×10⁻³小10×（数据集小） |
| 最小学习率 | 1×10⁻⁶ | Cosine退火下限 |
| 权重衰减 | 0.005 | 防过拟合 |
| 学习率调度 | CosineAnnealing | 标准选择 |
| Warmup | 10 epochs | 稳定初期训练 |
| 总Epochs | 150 | IP102够用；PlantVillage用100 |
| 批次大小 | 64 | 单卡3060可跑 |
| 标签平滑 | 0.1 | 防过拟合 |
| DropPath | 0.05 | 轻量模型用小值 |
| 混合精度 | 开启（FP16） | 加速训练 |

### 4.3 数据增强策略

**训练阶段**：

| 增强方法 | 参数 | 理由 |
|---------|------|------|
| RandomResizedCrop | scale=(0.7, 1.0) | 模拟不同拍摄距离 |
| RandomHorizontalFlip | p=0.5 | 标准增强 |
| RandomVerticalFlip | p=0.5 | 叶片翻转后病斑特征不变 |
| ColorJitter | brightness=0.2, contrast=0.2, saturation=0.1 | 模拟不同光照 |
| MixUp | alpha=0.2 | 对付类别不平衡 |
| CutMix | alpha=1.0 | 对付类别不平衡 |
| Normalize | ImageNet mean/std | 预训练权重要求 |

**测试阶段**：Resize(256) → CenterCrop(224) → Normalize(ImageNet mean/std)

### 4.4 消融实验（证明每个模块都有用）

**这是论文最核心的表格，必须完整。**

| 编号 | 模型配置 | 要验证的问题 |
|------|---------|------------|
| A | FasterNet-T0 原版（基线） | 基线性能 |
| B | A + MSCA（Stage3最后2个Block） | 注意力模块的贡献 |

| D | A + MSCA + 跨层融合（完整模型） | 两者组合的效果 |

**每组必须记录的指标**：

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| Top-1 Accuracy | 主要指标 | 预测概率最高的类是否正确 |
| Top-5 Accuracy | 补充指标 | 前5个预测中是否包含正确类 |

### 4.7 可视化实验（审稿人最爱看的部分）

**4.7.4 注意力权重可视化**（选做，加分项）

### 4.8 实验执行顺序（必须按此顺序）

```
第1步：跑通原版FasterNet-T0基线
  │     确认：能正常收敛，指标合理
  │
```

### 7.2 回复审稿意见的原则

1. **逐条回复**：审稿人的每一条意见都必须回应，不能遗漏
2. **态度谦虚**：即使审稿人理解错了，也要礼貌解释
3. **用数据说话**：补充实验来回应质疑，而不是争辩
4. **标注修改位置**：在回复中明确指出论文中修改了哪里（行号+页码）
5. **格式清晰**：审稿意见→你的回复→修改位置，三段式结构

### 7.3 常见审稿意见及应对

| 审稿人可能说的 | 你怎么回复 |
|--------------|-----------|
| "创新性不足" | 强调FasterNet首次应用于该领域；指出MSCA不是简单SE+DWConv，而是针对多尺度病斑的门控注意力 |
| "只用了2个数据集" | 承认局限性，但IP102是最大的害虫数据集，PlantVillage是最大的病害数据集，覆盖面已足够；未来将用自建数据集验证 |
| "没有和最新的方法比较" | 补充和1-2个2024-2025年方法的对比（如果审稿人指定了具体的） |
| "消融实验不够细" | 补充：MSCA中只用3×3 vs 只用5×5 vs 两者都用；reduction比的选择 |
| "实际部署效果呢" | 如果做了边缘部署实验，展示结果；如果没做，承认是未来工作 |
